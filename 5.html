3. 설계 판단

"대안이 있었지만 이걸 골랐다"
DECISION · 01

뒤로가기 가로채기를
네이티브 코드가 아닌 위젯 트리에서

상황
홈에서 안드로이드 뒤로가기 시 광고 + 종료 확인이 필요.
선택
WillPopScope + Future<bool> 다이얼로그
이유
화면 단위로만 적용 가능 / iOS·Android 단일 코드 / Navigator 컨텍스트 자연스럽게 활용.
감수한 비용
→ Flutter 1.x 호환성 한정. 신버전의 PopScope로 마이그레이션 부채 보유.
DECISION · 02

인터랙티브 카카오맵 대신
정적 이미지 폴백

상황
flutter_kakao_map은 비공식, iOS 미지원, libDaumMapAndroid.jar 수동 배치 필요.
선택
map.png 정적 이미지 + 주소 텍스트로 1차 가치 전달
이유
"기능이 부족한 것"보다 "빌드가 깨지는 것"이 더 큰 문제. MVP는 안정성이 우선.
감수한 비용
→ 핀치 줌·경로 검색 불가. 주석 처리된 import로 v2 재개 지점은 명시.
DECISION · 03

해시태그 탭은 UI만 두고
"내가 할거 아님" 명시

상황
검색 화면에 해시태그/지역명 두 모드가 요구되었으나 해시태그 색인은 백엔드 의존.
선택
탭 UI는 미리 만들고, 내용은 placeholder로 표시
이유
팀 분업 경계를 코드로 드러내, 화면 흐름 데모는 가능하면서도 책임 범위는 분명히.
감수한 비용
→ 데모 중 빈 탭이 보임. 대신 추정/속임수 구현으로 인한 향후 재작업 0.
